使用Protractor来测试Angular.js

发表于 2016-05-18   |   分类于 测试   |     |   阅读次数

端对端(e2e)测试

随着应用的复杂性和规模的增长，单纯的依赖手工测试来验证新的功能、捕捉错误、以及回归测试就变得不切实际。单元测试是捕获BUG的第一道防线。但是，有时组件之间的相互通信就很难靠单元测试来捕获异常。端对端(e2e)测试就是用来解决这样的问题的。
我们建立了Protractor，一个端对端测试来模拟用户的交互，这将有助于验证你的Angular应用的运行状况。

使用Protractor

Protractor是一个Node.js程序，所以运行端对端(e2e)测试需要编写JavaScript代码，并且使用Node。Protractor使用WebDriver（一种用于Web应用程序的自动测试工具）来控制浏览器，并模仿用户的行为。

  • Protractor get started
  • Protractor API

Protractor使用Jasmine作为他的测试语法。作为一个单元测试，一个测试文件一般会由一个或多个it块来描述你应用的需求。it模块是由命令和期望来组成的。命令告诉Protractor对应用来做一些事情，比如跳转页面，或者点击一个按钮。期望则是让Protractor断言一些应用程序的状态，比如某个字段的值是否正确，或者当前的URL是否正确。

当一个it块失败的时候，程序会把它标记为filed，并继续下一个模块的测试。

测试文件可能会有beforeEach和afterEach模块，分别运行在每个it模块之前，以及之后。而且不论当前it模块成功或者失败都会执行。

除了上述的内容，测试也可以包含一些辅助函数，以避免重复的代码。
