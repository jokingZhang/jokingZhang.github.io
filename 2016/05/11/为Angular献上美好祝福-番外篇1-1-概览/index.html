概念总览

本节使用一个简单的例子来描述AngularJS 所有重要的部分。如果想更加深入的了解，请参见教程

概念 描述
模板 Template 使用额外标记的HTML
指令 Directives 使用客户端属性（attribute）和元素（elements）扩展的HTML
模型 Model 当用户使用客户端时（发生交互时），在视图中显示的数据
作用域 Scope 数据存储的上下文，作用域可以被控制器（controllers），指令（directives），表达式（expressions）访问到。
表达式 Expressions 从作用域中访问变量（variables）和方法（Functions）
编辑器 Compiler 解析模板（template），实例化指令（directives）和表达式（expressions）
过滤器 Filter 使用者通过表达式来格式化用来展示的数据
视图 View 用户可以看到的（the Dom）
数据绑定 Data Banding 同步Model 和 view 之间的数据
控制器 Controller 视图背后的业务逻辑
依赖注射 Dependency Injection 创建并关联对象和函数
注射器 Injection 依赖注射的容器
模块 Module 配置注射器的容器，用来配置一个APP的方方面面，包括：控制器（Controllers）,服务（services）,过滤器（filters）和指令（directives）
服务 Service 独立于View的可重用的业务逻辑

这段看起来像是一段正常的HTML，但是可以看到一些新的标记。在AngularJS中，这样的文件被称之为模板。当Angular开始你的应用程序时，Angular会使用编辑器来解析并加工这些模板中的新标记。加载，转换，渲染DOM，之后才被称之为视图。第一种新标记被称为指令（directives），他们在HTML中为元素和属性申请了特殊的行为。在这个例子中，我们使用了ng-app属性，ng-app是自动初始化应用程序的指令。Angular同时也为input标签定义了指令来增加额外的行为。ng-model指令来存储，更新输入框中的变量。

自定义指令集来访问DOM：讲道理，在Angular中一个应用只能通过指令来访问DOM。这很重要，因为人工访问DOM将很难测试。如果你需要直接的访问DOM，你应该为此自定义指令。directives guide 告诉你如何去做。

第二种新的指令是双大括号

当编辑器遇到这个指令，将会使用新的标记来取代表达式中的值。在模板中的表达式（expression）是一种类似javascript代码片段的表达，并且允许Angular来读和写。请注意这些变量并不是全局变量。就像javascript中存在变量的作用域，Angular也为这些表达式提供了作用域。存储在作用域的变量的值被称为在文件的其余部分的模型。应用上边的例子，这些标记指导Angular来得到两数的乘积。
