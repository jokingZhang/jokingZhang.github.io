概念总览

本节使用一个简单的例子来描述AngularJS 所有重要的部分。如果想更加深入的了解，请参见教程

概念 描述
模板 Template 使用额外标记的HTML
指令 Directives 使用客户端属性（attribute）和元素（elements）扩展的HTML
模型 Model 当用户使用客户端时（发生交互时），在视图中显示的数据
作用域 Scope 数据存储的上下文，作用域可以被控制器（controllers），指令（directives），表达式（expressions）访问到。
表达式 Expressions 从作用域中访问变量（variables）和方法（Functions）
编辑器 Compiler 解析模板（template），实例化指令（directives）和表达式（expressions）
过滤器 Filter 使用者通过表达式来格式化用来展示的数据
视图 View 用户可以看到的（the Dom）
数据绑定 Data Banding 同步Model 和 view 之间的数据
控制器 Controller 视图背后的业务逻辑
依赖注射 Dependency Injection 创建并关联对象和函数
注射器 Injection 依赖注射的容器
模块 Module 配置注射器的容器，用来配置一个APP的方方面面，包括：控制器（Controllers）,服务（services）,过滤器（filters）和指令（directives）
服务 Service 独立于View的可重用的业务逻辑

这段看起来像是一段正常的HTML，但是可以看到一些新的标记。在AngularJS中，这样的文件被称之为模板。当Angular开始你的应用程序时，Angular会使用编辑器来解析并加工这些模板中的新标记。加载，转换，渲染DOM，之后才被称之为视图。第一种新标记被称为指令（directives），他们在HTML中为元素和属性申请了特殊的行为。在这个例子中，我们使用了ng-app属性，ng-app是自动初始化应用程序的指令。Angular同时也为input标签定义了指令来增加额外的行为。ng-model指令来存储，更新输入框中的变量。

自定义指令集来访问DOM：讲道理，在Angular中一个应用只能通过指令来访问DOM。这很重要，因为人工访问DOM将很难测试。如果你需要直接的访问DOM，你应该为此自定义指令。自定义指令 告诉你如何去做。

第二种新的指令是双大括号

当编辑器遇到这个指令，将会使用新的标记来取代表达式中的值。在模板中的表达式（expression）是一种类似javascript代码片段的表达，并且允许Angular来读和写。请注意这些变量并不是全局变量。就像javascript中存在变量的作用域，Angular也为这些表达式提供了作用域。存储在作用域的变量的值被称为在文件的其余部分的模型。应用上边的例子，这些标记指导Angular来得到两数的乘积。

上面的例子中还包含了过滤器。过滤器格式化了表达式中的值，并展示给用户。在上面的例子中，currency（货币）是的数字被格式化为货币的形式。

上面例子中最为重要的就是，Angular为我们提供了动态绑定，当输入变化时，表达式中的值会自动重新计算，同时，DOM也会根据这些值来刷新。这背后的概念是双向数据绑定

什么发生了改变？

首先，这里增加了一个包含了 controller (控制器) 的js文件。更确切的说，这个文件包含了一个创造了控制器实例的构造函数。控制器的目的是解析：变量，功能表达式（函数），指令。

除了在这个新文件中包含Controller的代码之外，我们还在HTML中添加了ng-controller指令。这个指令告诉Angular，这个新的InvoiceController控制器会对元素的指令，以及所有DOM的子节点负责。而语法InvoiceController as invoice告诉Angular实例化Controller，同时把它保存到一个叫做invoice的变量在当前的作用域。

同时，我们使用Controller的实例：invoice的前缀，改变了所有的页面中的表达式来读写变量。可能会出现的货币被定义在Controller中，并使用ng-repeat指令增加到了HTML模板中。同时，我们也可以使用

什么发生了改变？我们将之前定义的convertCurrency方法搬到了新文件finance2.js中。可是现在controller是怎么获得分离出去的方法呢？

这就是依赖注入的用武之地。依赖注入（DI）是一种设计模式，用来解决对象，函数如何创建，以及如何获取他们的依赖关系。所有Angular中的模块（directives，filters，controllers，services，…）使用依赖注入被创建和连接。在Angular中，DI的容器被称之为注射器。

使用依赖注入，需要有一个地方供所有东西被注册，工作。在Angular中，这就是module存在的目的。当Angular开始工作，它会使用module中的配置，由ng-app指令定义，包括这个模块在配置中依赖的所有模块。

上文中的例子：模板中包含了ng-app="invoice2"指令。他告诉Angular使用invoice2作为应用的主模块。而angular.module('invoice2', ['finance2'])则指定invoice2模块依赖finance2模块。通过这些，Angular 使用InvoiceController控制器 和 currencyConverter 来处理业务逻辑。

现在，Angular知道应用中的所有部分，并开始创建它们。在以前的部分，被使用factory方法创建。对于services，有很多方式来定义他们的factory。在上面的例子中，我们使用一个返回currencyConverter的方法作为service的工厂（factory）。

回到最初的问题，InvoiceController是如何得到currencyConverter函数的引用的？在Angular中，只需要在构造函数中定义简单的参数即可。在这个基础之上，注射器用正确的顺序创建对象，通过先前创建的，依赖的对象传进工厂。在我们的例子中，InvoiceController有名为currencyConverter的参数。通过这些，Angualr知道了controller和service之间的依赖关系，调用controller的同时，将service实例作为参数传递进去。

例子中最后有所改变的地方，就是module.controller现在使用的是一个数组，而不是一个简单的方法。数组首先会包含所有controller需要的依赖的service名称。数组中最后包含的是controller的构造函数。Angular使用这种数组的方式来定义，使得依赖注入在压缩代码后也可以正常工作，这将很可能使用更短的名称，比如a,来重命名构造函数。
